1. Зависит от масштабов использования генерации.
Если нужно сделать это один раз, и нет обязательности получать каждый раз новые строки, 
то проще создать массив строк один раз, чем разбираться в методах и возможностях библиотеки.
Если требуется использовать генерацию рандомных строк в нескольких местах + получать каждый раз новые строки и применять различные генераторы,
то, конечно, проще и удобнее использовать datafaker.

2. datafaker может пригодиться, исходя из вышенаписанного.
jfiglet - вряд ли, поскольку это просто украшалка вывода текста в консоль.

3. Указав ссылки из репозитория mavenCentral на эти библиотеки в build.gradle файле, мы подтягиваем эти библиотеки к себе в проект (gradle скачивает их jar-файлы из указанного репозитория)
